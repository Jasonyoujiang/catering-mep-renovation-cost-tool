# 餐饮商铺机电改造方案与成本测算工具设计

日期：2026-05-15

## 1. 项目目标

搭建一个面向商业项目餐饮商铺机电改造管理的轻量化本地 Web 工具，用于快速生成餐饮商铺机电改造方案参数，并查询常用机电子项的造价数据。

第一版重点不是追求计算规则一次到位，而是先建立清晰、可扩展的程序框架，便于后续逐步补充专业子项、计算逻辑、规格型号和价格资料。

工具默认服务于商管公司机电管理场景，输出应帮助管理人员快速判断：

- 改造条件是否基本匹配经营需求。
- 主要机电子项是否存在明显缺项。
- 初步成本是否具备决策参考价值。
- 后续深化设计、招采或施工阶段需要补充哪些资料。

## 2. 第一版范围

### 2.1 改造方案生成

用户输入：

- 商铺面积，必填。
- 餐饮类型，下拉选择：
  - 轻餐
  - 普通餐饮
  - 重油烟餐饮

程序输出第一批机电参数：

- 配套电缆规格。
- 供水管径。
- 排水管径。
- 排油烟风量。
- 占用隔油池容积。

第一版中的计算逻辑采用示例规则，作为框架验证用途。后续可根据项目经验、规范要求、供应商资料和成本数据库逐步替换为正式规则。

### 2.2 成本测算查询

用户操作：

- 第一级下拉菜单选择造价子项。
- 第二级下拉菜单根据第一级选择自动显示对应规格型号。
- 点击查询后输出造价信息。

第一版内置造价子项：

- 电缆。
- 不锈钢风管。
- 镀锌风管。
- 排油烟风机。

查询结果字段：

- 子项名称。
- 规格型号。
- 单位。
- 参考单价。
- 价格说明或备注。

第一版价格仅作为样例数据，用于验证交互和数据结构，不作为正式工程结算依据。

## 3. 暂不纳入第一版的内容

以下内容先预留接口，不在第一版展开：

- 登录、权限、多人协作。
- 数据库存储。
- Excel 或 CSV 导入导出。
- 复杂报表、正式概算书或清单计价文件。
- 与 CAD、BIM、ERP、招采系统联动。
- 按城市、时间、品牌、供应商自动匹配价格。
- 自动生成正式施工图设计说明。

这些内容后续可以在框架稳定后分阶段增加。

## 4. 技术方案

第一版采用轻量单页 Web 程序：

- `index.html`：页面结构。
- `styles.css`：界面样式。
- `app.js`：页面交互、输入校验、结果渲染。
- `data/items.js`：成本测算的子项、规格、价格数据。
- `data/rules.js`：改造方案计算规则与餐饮类型系数。

本方案的优点：

- 可直接在本地浏览器中打开使用。
- 不需要后端服务和数据库。
- 子项、规格和规则可以通过独立数据文件扩展。
- 适合先快速验证业务框架，再逐步升级为更完整系统。

## 5. 页面结构

页面分为两个主要工作区。

### 5.1 餐饮商铺机电改造方案生成

输入区：

- 商铺面积输入框。
- 餐饮类型选择框。
- 生成方案按钮。

输出区：

- 电气配置建议。
- 给排水配置建议。
- 排油烟配置建议。
- 隔油池容量建议。
- 风险提示。
- 后续需要补充的设计前提。

输出口径应保持工程管理可读性，采用“参数 + 判断 + 提醒”的表达，不只显示数字。

### 5.2 机电改造成本查询

输入区：

- 子项选择。
- 规格型号选择。
- 查询按钮。

输出区：

- 造价查询结果。
- 单价说明。
- 适用范围或注意事项。

当子项或规格未选择时，页面给出明确提示。

## 6. 数据与扩展设计

### 6.1 成本子项数据

成本数据采用数组或对象结构保存。每个一级子项包含多个规格型号。

每条规格数据建议包含：

- `id`：唯一标识。
- `name`：规格型号名称。
- `unit`：计量单位。
- `price`：参考单价。
- `remark`：价格说明。

后续新增子项时，只需要在 `data/items.js` 中增加一级分类和规格列表。

### 6.2 改造规则数据

改造方案计算规则独立放在 `data/rules.js` 中。

第一版规则按“面积 + 餐饮类型”计算，后续可扩展条件：

- 厨房面积。
- 就餐区面积。
- 是否明火。
- 是否火锅、烧烤、重油烟。
- 是否使用燃气。
- 原始电容量。
- 原始给水和排水条件。
- 原始排油烟井道条件。
- 是否具备隔油池接入条件。

每个专业子项应保留独立计算函数，避免所有逻辑写在一个大函数里。

## 7. 第一版示例规则原则

第一版可以采用保守的示例规则，主要用于验证系统框架：

- 轻餐采用较低用电、用水、排烟系数。
- 普通餐饮采用中等配置系数。
- 重油烟餐饮采用较高排油烟和排水隔油需求系数。
- 输出结果应标注“初步测算，需结合项目原始条件和深化设计复核”。

正式规则后续应根据以下资料逐步校准：

- 现行规范和地方要求。
- 商业项目历史餐饮改造经验。
- 设计院常用取值。
- 施工单位报价和供应商资料。
- 运营期投诉、能耗和维护数据。

## 8. 风险与合规提示

工具输出应默认带出以下提醒：

- 餐饮改造不能只按面积判断，还需核实原始供电容量、排烟井道、隔油池、排水接驳、消防条件和燃气条件。
- 排油烟系统需关注油烟净化、噪声、补风、防火阀、穿越防火分区等条件。
- 给排水系统需关注隔油、坡度、检修、反味、堵塞和运营维护。
- 电气系统需关注负荷容量、电缆路径、计量、保护开关和后期增容成本。
- 成本测算结果为管理测算口径，不替代正式设计概算、招标清单或结算审核。

## 9. 验收标准

第一版完成后，应满足：

- 本地打开页面可正常使用。
- 输入面积和餐饮类型后，可生成第一批机电参数。
- 成本测算两级下拉联动正常。
- 点击查询后能显示对应规格的造价信息。
- 新增一个成本子项时，不需要修改页面结构。
- 新增一个方案子项时，能通过独立规则函数接入。
- 页面文字为中文，表达适合商管机电管理人员使用。

## 10. 后续迭代方向

建议后续按以下顺序完善：

1. 补充正式计算规则和取值依据。
2. 扩展成本子项和规格型号库。
3. 增加“生成方案清单”或“导出报告”功能。
4. 增加项目原始条件输入，如原供电容量、井道条件、隔油池条件。
5. 增加风险分级，区分低风险、需复核、高风险。
6. 增加 Excel 或 CSV 数据导入，便于维护造价库。
7. 升级为可部署的小型 Web 应用。
